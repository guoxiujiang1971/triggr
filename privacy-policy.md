# Triggr Privacy Policy

**Last updated: July 17, 2026**

Thank you for using Triggr. This privacy policy explains how we collect, use, and protect your information.

## Information Collection and Use

The following information may be collected while using this app:

### Location
- Core functionality relies on geolocation to trigger safety reminders
- When a geofence trigger is configured, the app accesses your location **in the background** to detect entry/exit of designated areas
- Location data is stored **locally on device only** and is **never uploaded to any server**

### Bluetooth
- Used to detect Bluetooth beacons at entrances
- Only reads signal strength (RSSI) and UUID; no personal data is collected

### NFC
- Reads only the unique identifier (UID) of NFC tags for entrance detection
- No other data on the tag is read or stored

### WiFi
- Reads the current WiFi SSID locally to match a configured entrance network
- No passwords or network traffic data are collected

### Microphone
- Enabled only when you actively use the "record sound" feature
- Recorded audio files are stored locally on device

### User-Generated Content
- All scenarios, recordings, and settings are stored **entirely on device**
- The app **does not** collect, upload, or analyze any usage data

## Data Storage & Security

- All data resides in the device's local file system
- The app **has no remote server** and uses no cloud storage
- Uninstalling the app will erase all data; please back up important configurations beforehand

## Third-Party Services

This app **does not** use any third-party analytics SDKs, advertising SDKs, or data collection services.

## Children's Privacy

This app is not intended for children under 13, and does not knowingly collect any personal information from children.

## Changes to This Policy

If this privacy policy is updated, changes will be posted here. Continued use of the app constitutes acceptance of the updated policy.
